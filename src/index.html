Rails asset pipeline

Why not use it?

  • Namespacing sucks for large javascript apps
  • JS Packages install via npm
  • Frontend devs + designers use node tooling (grunt, gulp etc)

What should you use?

    • ES6 modules
    • Uses npm for package mgmt
    • Babel
    • Tree shaking
    • Hot module replacement

Options your left with

  • Abandon rails :(
  • rails-api + separate frontend
  • Add custom webpack scripts to your rails app
  • or…
  • Rails Productivity
  • rails-api caveats - SEO, slow JS boot times for news websites etc, overall complexity
  • Scripts – goes against rails mantra, convention over configuration

  • rails/webpacker

Useable with rails 5.0

Included with rails 5.1 (release TBC ~April 2017)

What you get…

  • $ rails new myapp --webpack
  • $ rails webpacker:install:vue (angular|react)
  • <%= javascript_pack_tag 'application' %>
  • hooks into $ rake assets:precompile
